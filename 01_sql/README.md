## Что внутри:
- **schema.sql** — структура таблиц и связи между нимиСУБД: MySQL 8.x (подойдёт и MariaDB 10.x, но тестировалось на MySQL)
- **seed.sql** — небольшой синтетический набор данных
- **queries** — запросы по темам (папкам), в каждой папке **1 файл = 1 запрос**

СУБД: MySQL 8.x (подойдёт и MariaDB 10.x, но тестировалось на MySQL)

## Таблицы
- **users** — пользователи (дата регистрации, канал привлечения, страна)
- **products** — товары (категория, цена)
- **orders** — заказы (дата создания, статус)
- **order_items** — состав заказа (товар, количество, цена на момент покупки)
- **payments** — оплаты (сумма, дата оплаты, метод, статус)
- **web_events** — события на сайте (визит, просмотр товара, добавление в корзину, оформление, покупка)
- **departments**, **employees** — отдельный блок для примеров оконных функций (ранжирование зарплат)

## Структура queries
- **01_users** — базовые запросы по пользователям (каналы, регистрации)
- **02_orders** — метрики по заказам и выручке (статусы, выручка по дням)
- **03_payments** — метрики по оплатам (успешность оплат, потери из-за failed)
- **04_joins** — примеры JOIN (LTV пользователя, выручка по категориям)
- **05_cte** — примеры CTE (воронка по событиям, сегментация по spend)
- **06_windows** — оконные функции (топ-3 по выручке в месяце, накопительная выручка, максимум зарплаты по отделам)

## Как запустить
1) Запустить **schema.sql** (создаст базу sql_portfolio_shop и таблицы)
2) Запустить **seed.sql** (заполнит таблицы данными)
3) Запускать запросы из папки **queries/** по одному файлу
