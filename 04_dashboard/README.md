# Дашборд: воронка, продажи и источники трафика (e-commerce)

## Цель
Показать в одном месте ключевые показатели интернет‑магазина:
- сколько пользователей проходит шаги воронки;
- как меняются продажи по дням;
- чем отличаются источники трафика по конверсии и выручке.

## Данные
Источник: **events_ecom.csv** (события пользователей)

Колонки:
- **user_id** — идентификатор пользователя  
- **event_time** — время события  
- **event_name** — тип события (visit, signup, add_to_cart, purchase)  
- **amount** — сумма покупки (заполнена только для purchase)  
- **source** — источник трафика (ads, email, organic, referral)

Период: октябрь–ноябрь 2025 (это видно по фильтру дат).

## Что есть на дашборде
### Ряд 1 — KPI
- **Посетители** — уникальные посетители (уникальные user_id с событием visit)
- **Покупатели** — уникальные покупатели (уникальные user_id с событием purchase)
- **Конверсия** — Buyers / Visitors
- **Доход** — сумма amount по покупкам
- **Покупки** — количество покупок (количество событий purchase)
- **AOV** — средний чек = Revenue / Purchases

### Ряд 2 — графики
- **Доход по дням** — выручка по дням
- **Покупки по дням** — количество покупок по дням
- **Воронка по пользователям:** — сколько уникальных пользователей дошло до каждого шага

### Ряд 3 — источники трафика
Таблица/срез по source:
- Посетители, Покупатели, Конверсия, Доход, Покупки, AOV

## Фильтры
- **Дата** — диапазон дат
- **Канал** — источник трафика

Фильтры применяются ко всем графикам на дашборде.
