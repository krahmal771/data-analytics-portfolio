# Портфолио: Data Analytics (SQL + A/B тесты + продуктовый кейс + дашборд)

В этом репозитории собраны учебные проекты, которые показывают мои базовые навыки аналитика: SQL-запросы, анализ A/B тестов, продуктовая аналитика (воронка/удержание) и дашборд в Tableau Public

---

## Инструменты
- SQL (JOIN, CTE, оконные функции)
- Python (pandas, базовая статистика)
- A/B тестирование (t-test, z-test долей, доверительные интервалы)
- Дашборд в Tableau Public
- GitHub для публикации проектов

---

## Навигация по папкам
- **01_sql** — SQL-проект (структура + данные + запросы)
- **02_ab_tests** — A/B тесты (2 проекта)
- **03_product_case_ecom** — продуктовый кейс (воронка/удержание)
- **04_dashboard** — дашборд (скриншот + описание)

---

## Подробнее про проекты

### 1) 01_sql — SQL-проект (e-commerce тематика)
**Цель:** показать умение писать запросы под типовые задачи аналитика

**Что внутри:**
- schema.sql — структура таблиц и связи
- seed.sql — небольшой набор данных для примеров
- queries — запросы по темам (1 файл = 1 запрос):
  - базовые метрики
  - JOIN
  - CTE
  - оконные функции

**Как проверять:**
1) Создать БД и выполнить schema.sql
2) Заполнить данными через seed.sql
3) Открывать папки в queries и запускать запросы по одному

---

### 2) 02_ab_tests — A/B тесты (2 мини-проекта)
**Цель:** показать практику проведения A/B теста под разные типы метрик

**Что внутри:**
- **ab_continuous_revenue** — непрерывная метрика (revenue):
  - загрузка и проверка данных
  - расчёт эффекта (delta)
  - статистический тест (t-test), p-value
  - доверительный интервал
  - проверка хвоста (p99) и устойчивость результата
- **ab_proportions_conversion** — доли (конверсия):
  - чистка данных (валидность соответствия группы/страницы)
  - расчёт конверсий, эффекта (delta)
  - z-тест долей, p-value
  - доверительный интервал
  - проверка по дням (daily delta, стабильность трафика)

Файлы в каждой папке:
  - **data** - файл с данными
  - **figures** - скриншоты графиков
  - **notebooks** - ноутбуки с тестами
  - **README** - описание проекта

---

### 3) 03_product_case_ecom — продуктовый кейс (e-commerce события)
**Цель:** показать умение работать с событийными данными и отвечать на продуктовые вопросы

**Что внутри:**
- EDA (размер, типы, пропуски, период, распределения)
- воронка по пользователям: visit → signup → add_to_cart → purchase
- retention по покупкам D7/D14

Файлы в папке:
  - **data** - файл с данными
  - **notebooks** - ноутбуки с тестами
  - **README** - описание проекта

---

### 4) 04_dashboard — дашборд
**Цель:** визуально показать основные показатели продукта на одной странице

**Что на дашборде:**
- KPI: Visitors, Buyers, Conversion, Revenue, AOV, Purchases
- Funnel (bar)
- Revenue daily (line)
- Purchases daily (line)
- таблица по источникам source

**Файлы:**
- **data** — файл с данными
- **screenshots** — скриншот дашборда
- **README.md** — описание дашборда
